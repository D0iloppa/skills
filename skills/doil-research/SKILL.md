# doil-research

외부 출처를 다수 수집해 종합 판단을 내려야 하는 **심층 리서치**를, 토큰 효율적으로 그리고
**중단·재개 가능한** 형태로 수행하는 방법론.

> 원칙: 가정하지 말 것·단순성 우선·수술적 변경·목표주도. 레포에 자체 에이전트 지침(CLAUDE.md 등)이
> 있으면 그것을 함께 따른다. 이 스킬은 "어떻게 리서치할지"의 절차이며, 개별 리서치의 산출물이 아니다.

## 착수 (Kickoff) — 처음 쓰는 사람을 위한 진입

**막연하게 지시해도 된다.** 사용자가 input 폴더 구조나 리서치 절차를 몰라도 되도록, 시작에 필요한
정보가 빠져 있으면 **수행자가 먼저 자연어로 물어 채운다**(임의로 가정하지 않는다 — "가정 금지" 원칙).

시작에 필요한 최소 정보와, 빠졌을 때 수행자의 처리:

| 필요한 것 | 예시 | 빠져 있으면 |
|-----------|------|-------------|
| **리서치 대상** | "부산 R&D 과제 116030" | 무엇을 조사할지 되묻는다 |
| **실제 의도**(결론이 답해야 할 질문) | "이 R&D로 급여를 태울 수 있나?" | 표면 지시에서 추론해 **제시하고 확인받는다** |
| **시드 자료**(URL·문서) | 링크, `input/`의 PDF | 있으면 받고, 없으면 수행자가 탐색으로 시작 |
| **작업공간 경로** | `_tmp/research/` | 기본값을 제안한다 |
| **스코프 경계** | "6번만, 나머지는 참고만" | 지시 범위로 한정할지 확인한다 |

수행자 행동 규약: 위에서 빠진 항목이 있으면 **한 번에 1~3개만 몰아 질문**하고 착수한다. 특히
**실제 의도**는 조사 심층도를 바꾸므로, 불명확하면 반드시 착수 전에 확정한다.

### 추천 지시 템플릿 (자연어 — 복붙해서 채우면 됨)

> **[대상]** 을 조사해줘. 진짜 알고 싶은 건 **[실제 의도 / 결론에서 답해야 할 질문]** 이야.
> 참고자료는 **[URL·파일 경로, 없으면 "없음"]**. 결과는 **[작업공간 경로]** 에 정리해줘.
> **[스코프: 어디까지 파고들지 / 무엇은 제외]**.

채운 예:

> "bizinfo 공고 116030(부산 Multi AI Agent R&D)을 조사해줘. 진짜 알고 싶은 건 *이 과제 인건비로
> 인력 급여를 실제로 조달할 수 있는가* 야. 참고자료는 `input/과제정리.pdf` 와 공고 URL. 결과는
> `_tmp/research/` 에 정리해줘. 6번 과제 중심으로, 나머지 공고는 참고만."

이 정도만 줘도 착수 가능하고, 덜 줘도 수행자가 위 표대로 되물어 채운다.

## 역할 규약

- **지시자**: 리서치를 요청한 사람(예: 차은우 대표님). 표면 지시 뒤의 **실제 의도**를 먼저 짚는다.
- **수행자**: 서강준(오케스트레이터). 계획·서브에이전트 지휘·종합·최종 판단을 담당.

리서치 착수 전 반드시 **표면 지시 vs 실제 의도**를 분리해 명시한다. (예: "6번 과제 조사" =
표면 / "그 과제 인건비로 급여를 태울 수 있는가" = 실제 의도) 실제 의도가 조사 초점·심층도를
바꾸므로 불명확하면 착수 전에 확인한다.

## 모델 역할분리 (근거 필수)

| 역할 | 모델 | 담당 | 근거 |
|------|------|------|------|
| 오케스트레이터 | **Opus** (메인 세션) | 계획 골격, 의도 해석, 종합, 최종 판단, 서브에이전트 지휘 | 판단·해석·종합은 추론 깊이가 필요 → Opus |
| 수집 서브에이전트 | **Sonnet** | URL 크롤링, raw 원문 다운로드, 표·첨부 추출, 1차 요약 | 정형화된 반복 처리 → 추론 깊이보다 처리량·비용효율 우선. Opus 쓸 이유 없음 |

- 서브에이전트는 `Agent` 툴로 **`model: "sonnet"`**, **`run_in_background: false`(동기·순차)** 로 띄운다.
- **모델을 고를 때마다 선택 근거를 한 줄로 남긴다**(위 표의 논리를 해당 단계에 맞게).

## 핵심 프로토콜

### 1. raw-first — 원문 먼저, 가공은 나중
- 서브에이전트는 **먼저 raw 텍스트를 저가토큰으로 수집**한다(WebFetch 원문 다운로드, PDF·표·첨부 원문).
  수집 단계에서 해석·재작성으로 토큰을 태우지 않는다.
- 수집물은 `sources/` 하위에 **원문 그대로** 저장한다. 재가공(요약·비교·판단)은 원문이
  파일로 확보된 뒤 별도 단계에서 한다. → 재수집 없이 몇 번이고 다시 가공 가능.

### 2. 순차 진행 — 병렬 금지
- 단계를 **한 번에 하나씩** 실행한다. 병렬로 여러 서브에이전트를 띄우지 않는다.
- 이유: 중간에 토큰이 소진돼도 **다음 세션에서 정확히 그 지점부터 이어받기** 위함.
- **각 단계 완료 직후 핸드오프 원장(`_HANDOFF.md`)을 갱신**한다. 이게 재개의 전제.

### 3. 핸드오프 원장 — md 파일 하나로 이어받기
- `_HANDOFF.md` **하나만 넘기면** 다른 세션이 문맥·진행상태·다음 착수지점을 복원할 수 있어야 한다.
- 정본은 **self-contained 일반 md**. (아래 "도구 경계" 참조 — 특정 MCP를 정본으로 삼지 않는다.)
- 템플릿: [`references/handoff-template.md`](references/handoff-template.md)

## 디렉토리 레이아웃

리서치 작업공간(지시자가 지정한 경로, 예: `_tmp/research/`) 하위:

```
<research_dir>/
  _HANDOFF.md               # 진행 원장(정본). 매 단계 후 갱신. 이 파일만 넘기면 재개 가능
  input/                    # 지시자가 준 입력물(PDF 등)
  sources/                  # 서브에이전트가 수집한 raw 원문 + 1차가공 (재가공의 기준)
  <주제>_조사.md            # 최종 종합 판단서(정본 산출물)
```

## 표준 단계 골격

```
1. (Opus) 정본 조사문서 골격 + _HANDOFF.md 생성
   → 검증: 조사질문·목차·의도 확정, 원장 초기화 완료
2..N. (Sonnet 서브에이전트) 출처별 raw 수집 — 한 번에 하나
   → 검증: sources/ 에 원문 저장, 원장에 [x] 체크
최종. (Opus) 종합·판단 — 원문 인용 + 출처 링크로 정본 문서 완성
   → 검증: 실제 의도에 대한 결론 명시, 근거 원문 링크 통합
```

각 단계는 **검증 기준을 먼저 정의**(목표주도)하고, 완료 시 원장에 반영한다.

## 서브에이전트 지시 방식

수집 서브에이전트에게는 다음을 명확히 전달한다:
- 대상 URL/파일 **그대로**(에이전트가 URL을 직접 WebFetch 하게 함)
- 저장 경로(`sources/<식별자>.md`)와 **raw 우선·해석 최소화** 원칙
- 뽑아야 할 항목 체크리스트(예: 지원규모·기간·인건비 인정범위·컨소시엄 요건·접수일정 등)
- "판단하지 말고 원문·수치·조항을 그대로 남겨라. 판단은 메인이 한다."

## 도구 경계

- **크롤링**: `WebFetch`(서브에이전트가 URL 직접 수신). 정형 데이터가 있으면 해당 환경의 도메인 MCP를 병용할 수 있다.
- **핸드오프 정본**: 일반 md 원장. **이유** — 다음 세션 프로필에 특정 MCP가 없어도 읽혀야 하고,
  "md 하나만 넘기면 착수"라는 전제를 지키려면 self-contained 여야 한다.
- **구조화 저장 MCP(선택적 보조, 예: djinn 등 그래프/KV)**: 수집한 출처·참조관계를 그래프(node/edge/meta)로
  색인해 "수집상태 질의"에 쓸 수 있으나, **핸드오프 정본으로 삼지 않는다**(그 MCP 없는 세션에서 재개 불가해지므로).
